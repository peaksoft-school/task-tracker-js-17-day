# 🎓 Школьный проект — Peaksoft House

Это учебный проект, разработанный в рамках курса. Основной упор — на практику работы с React, Redux Toolkit, современными UI-инструментами и правильной архитектурой компонентов.

---

## 🚀 Используемый стек

- **React** — основа пользовательского интерфейса  
- **Redux Toolkit** — управление глобальным состоянием и API-запросами  
- **Material UI (MUI)** — готовые UI-компоненты в едином стиле  
- **Storybook** — изолированная разработка и тестирование компонентов  
- **Architecture**: Component-Based (компонентно-ориентированная архитектура)

---

## ⚙️ Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск проекта в режиме разработки

```bash
npm run dev
```

---

## 🏁 Сборка и запуск в Production

Для подготовки production-сборки используйте:

```bash
npm run build
```

После сборки содержимое папки `dist` можно развернуть на сервере.

## 🌿 Работа с ветками Git (Git Branching Convention)

Для удобной навигации и командной работы принято использовать следующую структуру имен веток:

| Тип ветки     | Назначение                            | Пример                                |
|---------------|----------------------------------------|----------------------------------------|
| `feature/`    | Новая фича или функционал              | `feature/add-login-page`              |
| `bugfix/`     | Исправление бага                      | `bugfix/fix-navbar-hover`             |
| `refactor/`   | Рефакторинг без изменений функционала | `refactor/optimize-dashboard`         |
| `docs/`       | Работа с документацией                 | `docs/update-readme`                  |
| `test/`       | Добавление или изменение тестов       | `test/add-unit-tests`                 |
| `release/`    | Подготовка релиза                     | `release/v1.0.0`                      |

> ✅ Используйте `kebab-case` (через дефис) и англоязычные описания.

## 📂 Структура проекта (упрощённо)

```

```
